# 📝 Learning Log & Execution Tracker

> **Core Philosophy:** 
> **Rule #1:** Spend 40% of your time consuming theory and 60% building. 
> **Rule #2:** Consistency Over Hoarding. It is better to complete one module and build one project consistently than to bookmark 50 resources without implementation. Projects > Certificates.

---

## 🗓️ The Weekly Execution Model (The Blueprint)
*Follow this exact cadence for every phase and project to ensure production-readiness.*

| Days | Focus Area | Direct Actions | Expected Output |
| :--- | :--- | :--- | :--- |
| **Day 1-2** | **Conceptual Absorption** | Watch course videos at 1.25x speed. Read assigned book chapters. Take handwritten notes. | Completed notes + core concept understanding. |
| **Day 3-4** | **System Core Coding** | Close the tutorials. Write the project code from scratch. Type every line manually. | Working core implementation. |
| **Day 5-6** | **Debugging & Refactoring** | The "Suck" phase. Add Docker, set up MLflow/W&B/LangSmith, write `pytest` unit tests, fix env bugs. | Stable, tracked, and containerized code. |
| **Day 7** | **Deployment & Docs** | Write the `README.md`. Draw architecture diagram. Deploy to live URL. Post on LinkedIn. | Public, portfolio-ready repository. |

## ⏰ Daily Non-Negotiables Checklist
*Do not go to sleep without checking these off. Total time: ~5.5 hours.*

- [ ] **📖 Paper Reading / Engineering Blogs (30 min - Morning):** Read "Attention is All You Need", "LoRA", "RAG", or vendor release notes (Hugging Face, AWS, LangChain).
- [ ] **🧠 Theory / Coursework (2 hours - Mid-day):** Focused learning on the current phase content. No distractions.
- [ ] **💻 Coding / Project Work (3 hours - Evening):** Hands-on implementation. Typing manually, debugging, building.
- [ ] **📋 Review & Plan (15 min - Before bed):** Track progress, update the tracker below, and plan tomorrow's exact tasks.

### 💡 How to use this file effectively:
1. **Sunday Night Routine:** Spend 15 minutes every Sunday copying the **Weekly Progress Template** and filling in your goals for the upcoming week.
2. **Daily Accountability:** Check off your **Daily Non-Negotiables** every evening. If you miss a day, don't panic—just restart the streak the next day. Consistency beats intensity.
3. **The "Suck" Phase:** Days 5-6 are where most people quit because Docker, CI/CD, and async workers are frustrating. Use the **Bottleneck Troubleshooting Log** to document your errors. This exact log can later be turned into a highly impressive Medium/Dev.to blog post (e.g., *"How I Debugged Celery Worker Memory Leaks in Docker"*), which hiring managers love.
